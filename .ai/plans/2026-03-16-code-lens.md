## Goal

Add Code Lens support to rlsp-yaml so that when a YAML document
has an associated JSON Schema, the schema title (or URL) is shown
as a code lens at the top of the document. Clicking it opens the
schema URL. This gives users immediate visibility into which schema
is validating their file.

## Context

- `JsonSchema` already has a `title: Option<String>` field
- Schema associations are stored in `Backend::schema_associations`
  (maps document URI → schema URL string)
- Cached schemas are in `Backend::schema_cache` (maps URL → `JsonSchema`)
- The hover and completion handlers already demonstrate the pattern
  for accessing schema_associations → schema_cache with proper lock ordering
- tower-lsp 0.20 provides `code_lens` method on the `LanguageServer` trait
- `lsp_types` provides `CodeLens`, `CodeLensParams`, `CodeLensOptions`, `Command`
- The LSP `Command` has `title`, `command` (ID string), and `arguments`
- For "open URL", the standard command is `vscode.open` with the URL as argument
- This is a small, self-contained feature — single task slice

### Key files

- `src/code_lens.rs` — new module with pure function
- `src/server.rs` — wire up capability and handler
- `src/lib.rs` — add module declaration

### Patterns to follow

- Pure function: `code_lenses(schema_url, schema) -> Vec<CodeLens>`
- Capability: `CodeLensOptions { resolve_provider: Some(false) }`
- Lock ordering: `schema_associations → schema_cache`

## Tasks

### Task 1: Add code lens module and server integration

Add a new `src/code_lens.rs` module with a pure function that produces
a code lens showing the schema title, and wire it into the server.

**What to implement:**

A) `src/code_lens.rs` — pure function:
- `pub fn code_lenses(schema_url: &str, schema: Option<&JsonSchema>) -> Vec<CodeLens>`
- Produce a single `CodeLens` at line 0, col 0 (top of document)
- Title: schema title if available, otherwise the schema URL
- Command: `vscode.open` with the schema URL as argument
- Returns empty vec if no schema URL is provided (caller handles that)

B) `src/server.rs` — capability and handler:
- Add `code_lens_provider: Some(CodeLensOptions { resolve_provider: Some(false) })`
  to `capabilities()`
- Implement `code_lens` method on `LanguageServer`:
  - Get schema URL from `schema_associations`
  - Get cached schema from `schema_cache`
  - Call `code_lenses(url, schema)` and return result

C) `src/lib.rs` — add `pub mod code_lens;`

**Tests:**
- Code lens with schema title shows title
- Code lens without title shows URL
- No schema association returns empty vec
- Capability is advertised

## Decisions

- **Single code lens at line 0** — simple and non-intrusive, matches
  common code lens patterns in other LSP servers
- **`vscode.open` command** — standard VS Code command for opening URLs;
  other editors may not support this but it degrades gracefully (lens
  shows but click does nothing)
- **No resolve provider** — the code lens is fully resolved on first
  request; no need for a two-stage resolve
- **Single task** — this feature is small enough for one commit
